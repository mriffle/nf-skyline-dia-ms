# Correctness Audit: nf-skyline-dia-ms

**Date:** 2026-04-14
**Branch audited:** `pdc-batches` (commit `d159ac2`)
**Scope:** Full workflow correctness — input resolution, search engines, Carafe, Skyline/QC/packaging/upload, main.nf orchestration, schema/config.
**Method:** Five parallel explorations by subsystem, followed by direct re-verification of the highest-impact findings against the source.

Totals: **5 critical**, **7 moderate**, **5 minor**, 5 false positives rejected during verification.

---

## Critical (workflow-breaking or silently corrupts artifacts)

### C1. `proteowizard_version` and `dia_qc_version` are never captured from Skyline
**File:** `main.nf:67-68`, `main.nf:242-246`

Both are declared `null` at lines 67-68 and never reassigned. The Skyline workflow emits them (`workflows/skyline.nf:105,110`) but `main.nf` never does `proteowizard_version = skyline.out.proteowizard_version`. At line 242 they are passed into `search_engine_version.concat(proteowizard_version, dia_qc_version, carafe_version)` as raw nulls. `carafe_version` is reassigned in the same file (lines 201/205/208), so the omission is specifically for the Skyline-emitted versions.

**Outcome:** `nextflow_run_details.txt` is missing Skyline/msconvert/DIA-QC version metadata, and the concat may NPE depending on Nextflow's null tolerance.

**Fix sketch:** After the `skyline(...)` call (~`main.nf:240`), add `proteowizard_version = skyline.out.proteowizard_version` and `dia_qc_version = skyline.out.dia_qc_version`.

---

### C2. Minimize enabled ⇒ batch-document join fails
**File:** `workflows/skyline.nf:49`, `modules/skyline.nf:245`

Regex `/(_annotated)?\.sky$/` only strips `_annotated.sky` or `.sky`. After minimize, filenames become `{doc_name}[_batch]_minimized.sky.zip` or `{doc_name}[_batch]_annotated_minimized.sky.zip`. `baseName.replaceAll(...)` leaves `_minimized` (or `_annotated_minimized`) dangling. The expected key (line 48) is `{doc_name}[_batch]`, so the `.join(..., failOnMismatch: true)` on line 49 must fail.

**Outcome:** Any run with `skyline.minimize = true` breaks at batch-document assembly — QC, reports, hashes, and uploads all fail. No stub config exercises this combination, so CI does not catch it.

**Fix sketch:** Extend the regex to also strip `_minimized` before/after `_annotated`: `/(_annotated)?(_minimized)?\.sky$/` (or centralize via the `sky_basename` helper in `modules/skyline.nf:4-6`).

---

### C3. `skyline.fasta` Panorama auth check uses wrong parameter
**File:** `subworkflows/get_input_files.nf:29`

Line 29 reads `if(panorama_auth_required_for_url(params.fasta))` but the block downloads `params.skyline.fasta`. Three audit agents independently flagged this.

**Trigger:** `params.skyline.fasta` is a private Panorama URL and `params.fasta` is local (or vice versa).

**Outcome:** Either `file(params.skyline.fasta, checkIfExists: true)` fails on a URL at line 33, or `PANORAMA_GET_FASTA` is called with a local path at line 30. The auth-trigger coverage in `main.nf:336` is already correct — only the download-site decision is inverted.

**Fix sketch:** Change `params.fasta` to `params.skyline.fasta` on line 29.

---

### C4. Duplicate `exclude_projects` guard — `exclude_replicates` never gates normalization
**File:** `subworkflows/generate_qc_report.nf:13-14`

`run_impute_normalize()` checks `exclude_projects != null` twice. Line 14 was clearly intended to be `exclude_replicates`. Because `normalization_method` defaults to `'median'`, the default path still runs, so this is latent, but any user who explicitly sets `normalization_method = null` plus `exclude_replicates` will silently skip the filter/impute/normalize step.

**Fix sketch:** Change the second `params.qc_report.exclude_projects` to `params.qc_report.exclude_replicates`.

---

### C5. EncyclopeDIA + Carafe is unusable when no `params.spectral_library` is set
**Files:** `subworkflows/encyclopedia/main.nf:15-17`, `main.nf:195-200`

EncyclopeDIA validates `params.spectral_library` directly rather than the incoming channel. Carafe writes the library via `main.nf:200` as a channel override, but the param remains null. SPECIFICATION.md calls this out as a "quirk"; in practice it makes the combination invalid unless the user *also* supplies a dummy `params.spectral_library`.

**Decision required:** Either (a) reject the combination early in `main.nf`, or (b) change the EncyclopeDIA guard at lines 15-17 to test the channel contents instead of the param.

## Moderate (dropped artifacts, degraded behavior, confusing errors)

### M1. Final EncyclopeDIA combined `.elib` not included in `search_file_stats`
**Files:** `subworkflows/encyclopedia/main.nf:80,82`, `modules/encyclopedia.nf` (`ENCYCLOPEDIA_CREATE_ELIB`)

`output_file_stats` is sourced from per-file search stats only; the combined final elib is published but its md5/size never reach the checksum manifest or Panorama upload. Spec flags this gap (SPECIFICATION.md line 539) but the code has not been updated.

---

### M2. Final Cascadia `.blib` not included in `search_file_stats`
**File:** `subworkflows/cascadia/main.nf:45` (BLIB_BUILD_LIBRARY step)

Same pattern as M1: only per-file SSL hashes are emitted; the final blib is not. Spec flags this.

---

### M3. PDC batch-file parsing can crash on blank / truncated lines
**Files:** `subworkflows/get_pdc_files.nf:41-47`, `main.nf:358`

Both locations do `line.split('\t')[batch_idx]` with no bounds check. Groovy's split drops trailing empty fields, so a blank trailing line (common on hand-edited TSVs) causes `IndexOutOfBoundsException` when `batch_idx > 0`. Additionally, `if (fname && batch)` at line 45 silently skips rows with a blank batch column — the downstream "missing from batch file" error (line 89) then fires for that file, giving a misleading diagnosis.

**Fix sketch:** Skip blank lines before splitting; validate row length explicitly and error cleanly on short rows.

---

### M4. Carafe local-only inputs are not in the AWS auth trigger
**File:** `main.nf:332-344`

`is_panorama_authentication_required()` omits `carafe.peptide_results_file`, `carafe.carafe_fasta`, `carafe.diann_fasta`. The Carafe internals do accept Panorama URLs via `get_input_file`, so on AWS with any of these set to a private URL, the AWS secret is never built and the download fails at runtime.

**Decision required:** Either add these to the auth trigger, or reject Panorama URLs for these inputs at parameter-validation time.

---

### M5. Empty-string `pdc.batch_file` bypasses the `!= null` check
**File:** `main.nf:97,104`

`params.pdc.batch_file != null` passes for `""`, setting `use_batch_mode = true`, then `get_pdc_batch_names("")` calls `file("", checkIfExists:true)` for an unhelpful runtime error.

**Fix sketch:** Check for non-null **and** non-empty, or tighten the schema to reject empty strings.

---

### M6. Default Skyline template URL — staging not verified
**File:** `subworkflows/get_input_files.nf:58`

`file(params.default_skyline_template_file)` is passed a raw GitHub URL. Nextflow's `file()` does stage HTTP(S) URLs, so this works in practice, but there is no handling for network failure or cache misses. Flag for runtime verification rather than immediate fix.

---

### M7. DIA-NN silently accepts arbitrary-format spectral libraries
**File:** `subworkflows/diann/main.nf:58-60`

The conversion branch handles `.blib` and `.dlib`; anything else passes through unchanged. A user-supplied `.tsv`, `.speclib`, or other format is handed to DIA-NN as-is with no validation. If the file is not DIA-NN-compatible, the failure is deep inside DIA-NN rather than at input validation.

**Fix sketch:** Validate the library extension against a known-good list and either pre-convert or error at input time.

`params.search_engine.toLowerCase()` without `.trim()` means `"diann "` with trailing space falls through to the EncyclopeDIA format branch. The upstream normalization at `main.nf:71` already handles this for the dispatch, but Carafe re-reads the raw param.

## False positives rejected during verification

These were raised by audit agents but disproved by direct reading:

- **"PDC branch does not emit `file_json`, breaking metadata validation"** — PDC bypasses `get_replicate_metadata` entirely and uses PDC annotations directly (`main.nf:177-189`). No validation gap.
- **"`save_run_details` is called in two branches"** — Only one branch runs per execution (`msconvert_only` `return` at `main.nf:171`); not a duplication risk.
- **"Deprecated Skyline params silently diverge"** — `check_old_param_name()` rewrites `params.skyline.*` atomically at `main.nf:45-52` before any downstream read. No divergence.
- **"Replicate metadata Panorama Public miswired"** — The spec's "miswired" note is stale; current code routes private vs public correctly.
- **"Normalization guard fails when `exclude_replicates` is set alone"** — Real, but upstream defaults make it latent. Reclassified above as C4, not rejected outright.

## Not covered by this audit (worth a follow-up)

- **Interaction of `use_vendor_raw` with each search engine** — Cascadia expects mzML; if `use_vendor_raw` is set with Cascadia the upstream resolver leaves RAW/`.d` as-is, and there is no explicit guard. DIA-NN handles vendor RAW; EncyclopeDIA semantics unclear.
- **Concurrent-batch correctness** — DIA-NN batch support was not stressed for cross-batch contamination (shared stage-in paths, duplicate batch names, batch-name collisions with reserved strings).
- **Panorama upload ordering when `panorama.import_skyline=true`** — agents checked the guard exists but did not trace the actual ordering enforcement.
- **Process caching directives** — Carafe lacks `storeDir`; in long runs this may redo large work unnecessarily.
- **Schema vs. code drift audit at scale** — agents spot-checked; a full param-by-param comparison of `nextflow_schema.json` against every reference in `main.nf`/workflows/subworkflows was not performed.

---

## Recommended next steps

1. Fix **C1** and **C2** first — both break core flows (version tracking and minimize). C2 will fire on any production run with minimize enabled.
2. Fix **C3** and **C4** as one-line typo corrections.
3. Decide policy on **C5** (Carafe + EncyclopeDIA): reject the combination in `main.nf` or change the EncyclopeDIA guard to test the channel.
4. Add stub-run configs for: `skyline.minimize=true`, Carafe + EncyclopeDIA, empty-string `pdc.batch_file`, blank-line PDC batch TSV, `params.skyline.fasta` Panorama URL with local `params.fasta`. None of these are in the current matrix.
5. Extend `search_file_stats` to cover the final combined elib and Cascadia blib (**M1**, **M2**) so the checksum manifest is complete.
